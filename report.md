# OutboundHumanlike-Bench：AI 外呼拟人度评测框架 · 技术报告 v1.0

**覆盖行业：** 电商 / 金融 / 保险 / 医疗健康  
**覆盖坐席：** 销售坐席 / 客服坐席（通知型 · 咨询解答型 · 回访型）

---

## 摘要

本项目提出一套面向 AI 外呼场景的拟人度评测框架，覆盖销售坐席与客服坐席（含通知型、咨询解答型、回访型三类），支持电商、金融、保险、医疗健康四个行业。框架的三项核心贡献：

- 基于 MirrorBench 风格 LLM-as-a-Judge 的场景专项评分体系，针对销售与客服坐席分别设计专属评分维度；
- 系统性 LLM Judge 去偏机制，消融实验验证去偏后评分区分度显著提升，证明去偏设计是框架有效性的前提；
- 真人录音多层排序验证，通过引入真实外呼录音数据，验证 Benchmark 对不同拟人度层级的排序能力。

所有数据、代码与评测工具完整开源。

## 一、核心评测结果

### 1.1 两类坐席总体得分

**本框架**：去除了 LLM Judge 系统性偏见，含 Veto 一票否决机制、五类偏见压制指令及场景专项标准。

**消融对照组**：含 LLM Judge 固有偏见，仅包含维度名称与打分说明，不做任何偏见干预。

**客服坐席**（通知型 42% / 咨询解答型 33% / 回访型 25%，按数据量加权）

| 评测对象 | 本框架（去除 LLM Judge 偏见） | 消融对照组（含 LLM Judge 偏见） |
|---------|:----------------------------:|:-----------------------------:|
| 基线模型（无拟人指令） | 0.36 | 1.93 |
| AI 风格对齐 | 1.95 | 1.98 |
| 真人录音（最佳） | 1.95 | 1.90 |

**销售坐席**

| 评测对象 | 本框架（去除 LLM Judge 偏见） | 消融对照组（含 LLM Judge 偏见） |
|---------|:----------------------------:|:-----------------------------:|
| 基线模型（无拟人指令） | 0.59 | 1.88 |
| AI 风格对齐 | 1.93 | 1.99 |
| 真人录音（最佳） | 1.89 | 1.83 |

**这张表说明了什么：**

1. **AI 风格对齐数据质量高，接近真人水平。** AI 风格对齐数据是通过拟人化要求指令生成的话术，在两类坐席中均达到 1.93–1.95 分，与真人最佳录音（1.89–1.95）高度接近，说明本框架的对话生成 Prompt 能有效模拟真人外呼的语言特征。

2. **Benchmark 对三类数据的排序有效且合理。** 基线模型（0.36 / 0.59）、AI 风格对齐（1.95 / 1.93）、真人最佳录音（1.95 / 1.89）三层得分呈现清晰的梯级分布，排序方向与预期完全一致，证明 Benchmark 能有效区分不同拟人度层级的数据，并对其进行合理排序。销售坐席真人（1.89）略低于 AI 风格对齐（1.93），原因是大学生录音人员在转化果断度、多轮挽回韧性等销售专项维度上缺乏实战经验，而 AI 风格对齐数据经专项 Prompt 精心设计，在这些维度上能稳定达到高分。

3. **去偏机制是框架有效性的前提。** 消融对照组去掉去偏设计后，基线得分大幅上升（1.93 / 1.88），与 AI 风格对齐几乎持平——未经去偏的 LLM Judge 会系统性地给「结构完整、表达礼貌」的内容打高分，而这恰恰是 AI 机器味的典型特征。

## 二、实验设计

### 2.1 整体流程

```
数据生成
  ├── 千问生成基线对话（无拟人指令）
  └── 千问生成风格对齐对话（拟人化 Prompt）

真人录音
  ├── 大学生扮演坐席，豆包语音扮演客户
  ├── 豆包实时转录（含口语保留指令）
  └── 人工校对：还原被语音模型省略的口语内容
          ↓ 质量控制：同场景取最高分，剔除 Veto 命中条目

评测
  └── Gemini Flash（跨厂商，避免模型偏好自身输出）
          ↓ 场景路由：销售坐席 / 客服坐席三子类
          ↓ MirrorBench 风格评分 + 去偏机制 + Veto 机制
          ↓ 维度得分 + 总分 + Veto 统计
```

### 2.3 LLM Judge 去偏机制与消融实验

#### 去偏机制设计

本框架采用 **MirrorBench 风格**的 LLM-as-a-Judge 评分体系，基于四个核心设计原则：

**原则一：Critique-then-Score（批判先于打分）**

这是本框架去偏的核心机制。评分模型被要求在给出最终分数前，必须强制完成三个分析步骤：

1. **上下文分析**：理解对话所处阶段、用户真实意图与情绪基调
2. **机器味挑刺**（AI Smell Critique）：主动识别所有 AI 风格特征，判断是否触发 Veto
3. **推理说明**：解释为什么给出这个分数，不允许直接跳到结论

这三步强制分析形成对评分结果的交叉校验（Cross Validation），确保最终分数基于多维度的批判性分析，而非 Judge 模型的直觉偏好。

**原则二：五类偏见显式压制**

在评分 Prompt 中以「反偏见警告」的形式显式告知 Judge 模型其固有偏见，并给出正确的替代标准：

| 偏见类型 | Judge 的错误倾向 | 本框架的纠正方式 |
|---------|--------------|--------------|
| 助人偏见（最危险） | 解答详细即得高分，不管是否像真人 | 明确声明「拟人度」与「有用性」是两个维度，即使完美解决问题，只要像 AI 就必须给 0 分 |
| 长度偏见 | 回复越长被认为越详细越好 | 简短克制列为正向标准，冗长啰嗦扣高分 |
| 礼貌偏见 | 过度客套被认为专业 | 开头结尾套话列为 Veto 触发条件 |
| 结构偏见 | 1234 条目化被认为清晰 | 条目化直接触发 Veto，一票否决 |
| 书面语偏见 | 书面语被认为规范正式 | 口语化句法列为满分标准 |

**原则三：Few-Shot 真实锚定**

评分 Prompt 中嵌入真实 CCO 话术的正反对比示例，强制 Judge 模型以真实人类表达为参照系，而非以 AI 生成的「完美答案」为参照系。示例明确展示：带有「呃」「嗯」「稍等哈」等语气词的不完美表达应得 2 分，而逻辑清晰但带播音腔的回复应得 0 分。

**原则四：Veto 一票否决机制**

当检测到以下任何一种 AI 特征时，该条对话总分直接归零：

| 触发类别 | Veto 条件示例 | 适用场景 |
|---------|------------|---------|
| 条目化结构 | 使用「1. 2. 3.」或「第一 / 第二 / 第三」 | 全场景 |
| 礼貌套话 | 开头「非常感谢您的咨询」/ 结尾「祝您生活愉快」 | 全场景 |
| 书面语堆砌 | 「我们将竭诚为您服务」「根据您的问题」 | 全场景 |
| 播音腔开场 | 开场前 20 秒无任何语气词与自然停顿 | 通知型 |
| 单轮超长 | 单轮回复超过字数上限 | 咨询解答型 |
| 信号误判 | 负面反馈出现后走正面应答路径 | 回访型 |
| 啰嗦惩罚 | 客户已释放购买信号后继续背稿介绍 | 销售型 |
| 机械重复 | 连续两轮使用相同角度挽回 | 销售型 |

### 2.4 真人数据采集说明

本项目由大学生扮演客服与销售坐席，使用豆包语音与 AI 扮演的客户实时对话，豆包同步进行语音转录。

采用这种数据采集方式的原因有两点：**一是成本低**，无需聘请专业坐席人员；**二是行业覆盖面广**，真实外呼录音数据通常受隐私保护，难以公开获取，本方法可灵活覆盖电商、金融、保险、医疗健康等多个行业场景。

豆包作为端到端语音模型，在转录时会本能地将口语内容文本化——省略「嗯」「哈」「就是那个」等语气词与停顿。这类省略并非识别错误，而是模型的文本化倾向，会改变对话的拟人度特征。因此，所有转录文本均经过人工校对，还原口语内容后再参与评测。

| 项目 | 说明 |
|------|------|
| 录制人员 | 2 名大学生，扮演客服 / 销售坐席 |
| 客户角色 | 豆包语音扮演（按场景给定角色指令） |
| 数据规模 | 164 条（电商 48 / 金融 40 / 保险 40 / 医疗健康 36） |
| 质量控制 | 同行业 × 类型 × 客户状态取最高分，剔除 Veto 命中条目 |
| 最终有效条数 | 49 条最佳录音（覆盖四行业四类场景） |

## 四、局限性

| 问题 | 说明 |
|------|------|
| 保险销售坐席区分度偏低 | 模型训练数据中保险销售话术丰富，基线模型本身输出质量已接近拟人水平 |
| 真人数据覆盖场景有限 | 真人最佳录音仅 49 条，用于定性验证排序，非核心统计数据 |
| 医疗健康真人得分偏低 | 情绪感知要求高的场景对非专业录音人员存在较高挑战 |
| 真实业务数据尚未引入 | 当前使用模拟数据，后续计划接入脱敏真实外呼录音进行校验 |
